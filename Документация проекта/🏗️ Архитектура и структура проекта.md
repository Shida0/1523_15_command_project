## 🌟 Общая архитектура

### Архитектурный стиль

Проект использует **доменно-ориентированную архитектуру (Domain-Driven Design)** с элементами **чистой архитектуры (Clean Architecture)**.

**Ключевые принципы:**
- **Разделение ответственности** — каждый компонент делает одну вещь
- **Изоляция доменов** — астероиды, сближения и угрозы независимы
- **Общая инфраструктура** — повторяющийся код вынесен в `shared/`
- **Инверсия зависимостей** — бизнес-логика не зависит от деталей реализации

## 🔧 Компоненты системы

### 1. API Layer (api/)

**Назначение:** HTTP эндпоинты для фронтенда

**Структура:**
```python
# Пример: api/asteroid_api.py
@router.get("/asteroids/all")
async def get_all_asteroids(
    skip: int = 0,
    limit: int = 100
):
    """Получить все астероиды"""
    service = get_asteroid_service()
    return await service.get_all(skip, limit)
```

**Ответственность:**
- Приём HTTP запросов
- Валидация параметров
- Вызов сервисов
- Возврат JSON ответов

### 2. Domain Layer (domains/)

**Назначение:** Бизнес-логика приложения

**Структура каждого домена:**

```
asteroid/
├── models/          # Таблицы базы данных
├── schemas/         # Pydantic схемы для валидации
├── repositories/    # CRUD операции с БД
└── services/        # Бизнес-правила
```

**Пример модели:**
```python
# domains/asteroid/models/asteroid.py
class AsteroidModel(Base):
    __tablename__ = "asteroids"
    
    id = Column(Integer, primary_key=True)
    designation = Column(String, unique=True, index=True)
    name = Column(String, nullable=True)
    estimated_diameter_km = Column(Float)
    earth_moid_au = Column(Float)
    orbit_class = Column(String)
    # ... другие поля
```

**Пример сервиса:**
```python
# domains/asteroid/services/asteroid_service.py
class AsteroidService(BaseService):
    async def get_by_designation(self, designation: str):
        """Получить астероид по обозначению"""
        async with UnitOfWork(self.session_factory) as uow:
            asteroid = await uow.asteroid_repo.get_by_designation(designation)
            return self._model_to_dict(asteroid)
```

### 3. Shared Layer (shared/)

**Назначение:** Общие компоненты для всех доменов

#### Config Manager

**Файл:** `shared/config/config_manager.py`

**Назначение:** Управление конфигурацией приложения

```python
class ConfigManager:
    def __init__(self):
        self.database = DatabaseConfig()
        self.nasa_api = NasaApiConfig()
        self.application = ApplicationConfig()
```

#### External API Clients

**Файлы:** `shared/external_api/clients/*.py`

**Назначение:** HTTP клиенты для NASA API

**Клиенты:**
- `SBDBClient` — Small-Body Database (астероиды)
- `CADClient` — Close Approach Data (сближения)
- `SentryClient` — Sentry Risk Table (угрозы)

#### Resilience Patterns

**Файлы:** `shared/resilience/*.py`

| Паттерн | Файл | Назначение |
|---------|------|------------|
| **Circuit Breaker** | `circuit_breaker.py` | Защита от сбоев API |
| **Bulkhead** | `bulkhead.py` | Ограничение одновременных запросов |
| **Timeout** | `timeout.py` | Таймаут запросов |

**Пример использования:**
```python
@circuit_breaker(NASA_API_CIRCUIT_CONFIG)
@bulkhead(CAD_BULKHEAD_CONFIG)
@timeout(NASA_API_TIMEOUTS['cad'])
async def get_close_approaches(...):
    # Запрос к API защищён от сбоев
    pass
```

#### Unit of Work

**Файл:** `shared/transaction/uow.py`

**Назначение:** Управление транзакциями

```python
async with UnitOfWork(session_factory) as uow:
    # Все операции в одной транзакции
    asteroid = await uow.asteroid_repo.create(data)
    approaches = await uow.approach_repo.bulk_create(list)
    await uow.commit()  # Или откат при ошибке
```

## 🔄 Потоки данных

### Поток 1: Получение данных из NASA

```
1. UpdateService запускается по расписанию
   │
   ├─▶ 2. SBDBClient запрашивает астероиды
   │   ├─▶ Circuit Breaker проверяет доступность
   │   ├─▶ Bulkhead ограничивает поток
   │   └─▶ Timeout прерывает при задержке
   │
   ├─▶ 3. AsteroidService обрабатывает данные
   │   ├─▶ Валидация через Pydantic схемы
   │   ├─▶ Расчёт недостающих параметров
   │   └─▶ Локализация (перевод на русский)
   │
   └─▶ 4. AsteroidRepository сохраняет в БД
       ├─▶ Проверка на дубликаты
       ├─▶ INSERT или UPDATE
       └─▶ Commit транзакции
```

### Поток 2: Запрос данных фронтендом

```
1. Пользователь открывает страницу
   │
   ├─▶ 2. JavaScript вызывает api.getAllAsteroids()
   │
   ├─▶ 3. Браузер отправляет GET /asteroids/all
   │
   ├─▶ 4. FastAPI принимает запрос
   │   ├─▶ Валидация параметров (skip, limit)
   │   └─▶ Вызов AsteroidService
   │
   ├─▶ 5. AsteroidService запрашивает данные
   │   └─▶ AsteroidRepository → SELECT из БД
   │
   └─▶ 6. JSON ответ возвращается в браузер
       └─▶ JavaScript рендерит таблицу
```

## 📊 Масштабирование

### Горизонтальное масштабирование

**API слой:**
- Запуск нескольких воркеров Uvicorn
- Балансировка нагрузки через Nginx

**База данных:**
- Репликация PostgreSQL
- Connection pool для эффективности

### Вертикальное масштабирование

**Оптимизация запросов:**
- Индексы на полях поиска
- Bulk-операции для записи
- Кэширование частых запросов

## 🔐 Безопасность

### Защита данных

- **Конфигурация** хранится отдельно от кода
- **Пароли** не хардкодятся в коде
- **API ключи** не требуются (NASA API публичный)

### Rate Limiting

- Задержка между запросами к NASA: 2 секунды
- Максимум запросов в час: 1,000
- Автоматическая пауза при превышении
