# 🌌 API документация Asteroid Watch

Добро пожаловать в API системы мониторинга астероидов **Asteroid Watch**!  
Этот документ поможет вам быстро разобраться с доступными эндпоинтами, форматами запросов и ответов. Все примеры можно скопировать и выполнить через терминал (curl) или использовать в вашем фронтенд-приложении.

---

## 📡 Базовые сведения

### Базовый URL
```
http://localhost:8000
```
Если сервер запущен на другом хосте/порте, замените соответственно.

### Формат ответов
Все ответы приходят в формате **JSON**. Успешные запросы возвращают код `200 OK`. В случае ошибки вы получите JSON с описанием проблемы.

### Пагинация
Большинство эндпоинтов, возвращающих списки, поддерживают параметры пагинации:
- `skip` — количество пропускаемых записей (по умолчанию `0`)
- `limit` — максимальное количество записей в ответе (по умолчанию `100`, максимальное значение может варьироваться)

Пример: `/asteroids/all?skip=20&limit=10` — пропустить 20 записей и вернуть следующие 10.

### Обработка ошибок
В случае ошибки сервер возвращает соответствующий HTTP‑статус и JSON вида:
```json
{
  "detail": "Описание ошибки"
}
```
Возможные статусы:
- `200 OK` — успех
- `400 Bad Request` — неверные параметры запроса
- `404 Not Found` — объект не найден
- `422 Unprocessable Entity` — ошибка валидации параметров
- `500 Internal Server Error` — внутренняя ошибка сервера

## 🪨 Эндпоинты астероидов (`/asteroids`)

### 1. Получить околоземные астероиды (PHA)
**GET** `/asteroids/near-earth`

Возвращает список астероидов, чьё минимальное расстояние пересечения орбиты Земли (MOID) меньше заданного.

#### Параметры запроса
| Параметр | Тип | Обязательный | По умолчанию | Описание |
|----------|-----|--------------|--------------|----------|
| `skip` | integer | нет | 0 | Количество пропускаемых записей |
| `limit` | integer | нет | 100 | Максимальное количество записей (≤1000) |
| `max_moid` | float | нет | 0.05 | Максимальное значение MOID в а.е. (0.0–0.5) |

#### Пример запроса
```bash
curl "http://localhost:8000/asteroids/near-earth?max_moid=0.03&limit=5"
```

#### Пример ответа
```json
[
  {
    "id": 1,
    "created_at": "2024-03-15T10:30:00",
    "updated_at": "2024-03-15T10:30:00",
    "designation": "433",
    "name": "Eros",
    "perihelion_au": 1.133,
    "aphelion_au": 1.783,
    "earth_moid_au": 0.028,
    "absolute_magnitude": 11.16,
    "estimated_diameter_km": 16.84,
    "accurate_diameter": true,
    "albedo": 0.25,
    "diameter_source": "measured",
    "orbit_id": "123456",
    "orbit_class": "Amor"
  }
]
```

## 📘 Пояснения терминов

- **MOID** (Minimum Orbit Intersection Distance) — минимальное расстояние между орбитами астероида и Земли. Чем меньше значение, тем выше потенциальная опасность.
- **Туринская шкала** — от 0 до 10, оценивает опасность столкновения: 0 — нет риска, 5–10 — серьёзная угроза.
- **Палермская шкала** — логарифмическая шкала, сравнивающая вероятность столкновения с фоновым уровнем. Отрицательные значения означают ниже фона.
- **Категория воздействия** — оценка последствий столкновения:
  - **локальный** — разрушения в радиусе десятков км
  - **региональный** — разрушения в масштабах страны/континента
  - **глобальный** — катастрофические изменения климата
