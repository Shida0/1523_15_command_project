## 🌟 Что такое фронтенд?

**Фронтенд** — это всё, что вы видите в браузере: кнопки, таблицы, анимации, текст. В нашем проекте фронтенд отвечает за отображение данных об астероидах, сближениях и угрозах в красивом и удобном виде.

### Технологии
- **HTML** — структура страниц (как каркас здания)
- **CSS** — внешний вид (цвета, шрифты, отступы)
- **JavaScript** — «оживление» страниц (загрузка данных, реакция на клики)

**Важно:** Мы не используем сложные фреймворки типа React или Vue. Всё написано на чистом JavaScript — это проще для понимания и поддержки.

## 📡 Как данные попадают на страницу?

### api.js — наш «почтальон»

Этот файл отвечает за связь с сервером. Представьте, что это курьер, который бегает на склад (сервер) и приносит данные.

**Базовый адрес сервера:**
```javascript
http://localhost:8000/api/v1
```

### Пример работы «почтальона»

```javascript
// Запросить все астероиды
const asteroids = await api.getAllAsteroids(0, 100);

// Запросить статистику
const stats = await api.getAsteroidsStatistics();
```

### Доступные «услуги почтальона»

#### 🪨 Астероиды
| Функция | Что делает |
|---------|------------|
| `getAllAsteroids()` | Принести все астероиды |
| `getNearEarthAsteroids()` | Только околоземные (опасные) |
| `getAsteroidDetails()` | Подробно об одном астероиде |
| `getAsteroidsStatistics()` | Статистика (сколько всего, средние размеры) |

#### 🌍 Сближения
| Функция | Что делает |
|---------|------------|
| `getUpcomingApproaches()` | Ближайшие сближения |
| `getClosestApproaches()` | Самые близкие пролёты |
| `getApproachesInPeriod()` | Сближения за период |

#### ⚠️ Угрозы
| Функция | Что делает |
|---------|------------|
| `getCurrentThreats()` | Все текущие угрозы |
| `getHighRiskThreats()` | Только опасные (Туринская шкала ≥ 5) |
| `getThreatsStatistics()` | Статистика угроз |

## 🏠 Главная страница (`index/`)

### Что на ней есть?
1. **Герой-секция** — красивый заголовок с описанием
2. **Статистика** — 4 карточки с цифрами (сколько астероидов, сближений, угроз)
3. **Таблица** — первые 15 астероидов для примера
4. **Туринская шкала** — интерактивная шкала опасности

### Как работает?

**Шаг 1:** Страница загружается в браузере

**Шаг 2:** JavaScript запускает функцию `loadPageData()`

**Шаг 3:** Функция делает два дела параллельно:
- Запрашивает статистику → обновляет карточки
- Запрашивает астероиды → заполняет таблицу

**Шаг 4:** Пользователь видит готовую страницу

### Пример данных астероида

```javascript
{
    designation: "433",           // Обозначение
    name: "Eros",                 // Имя (если есть)
    estimated_diameter_km: 16.84, // Диаметр в километрах
    earth_moid_au: 0.028,         // Насколько близок к Земле
    orbit_class: "Amor",          // Тип орбиты
    threat_assessment: {
        ts_max: 0,                // Туринская шкала (0-10)
        energy_megatons: 125.0    // Энергия удара в мегатоннах
    }
}
```

## 🪨 Каталог астероидов (`asteroids/`)

### Что на ней есть?
- **Полная таблица** всех астероидов (тысячи записей)
- **Фильтры** — отобрать по риску, диаметру, типу орбиты
- **Сортировка** — по имени, размеру, опасности
- **Пагинация** — разбивка на страницы по 50 штук

### Как работает фильтрация?

```javascript
// Пользователь нажал «Высокий риск»
filterByRisk('high');

// JavaScript фильтрует данные:
// 1. Берёт все астероиды
// 2. Оставляет только с TS >= 2
// 3. Показывает отфильтрованные
```

### Доступные фильтры

| Фильтр | Варианты | Что делает |
|--------|----------|------------|
| **Риск** | Все, Высокий, Околоземные | По уровню опасности |
| **Диаметр** | Измерен NASA, Расчётный | По точности данных |
| **Орбита** | Apollo, Aten, Amor | По типу орбиты |

## 🔍 Детали астероида (`asteroid_detail/`)

### Что на ней есть?
1. **Основная информация** — имя, размер, источник данных
2. **Орбитальная информация** — перигелий, афелий, MOID
3. **Оценка угрозы** — Туринская и Палермская шкалы, вероятность
4. **Сближения** — когда и насколько близко пролетит

### Как попасть на страницу?

В адресной строке браузера:
```
asteroid_detail.html?name=433
```

**`name=433`** — это параметр, который говорит странице: «Покажи астероид с номером 433»

### Как работает?

```javascript
// 1. Страница читает параметр из URL
const designation = "433";

// 2. Запрашивает данные с сервера
const asteroid = await api.getAsteroidDetails(designation);

// 3. Заполняет все поля на странице
document.getElementById('diameter').textContent = asteroid.estimated_diameter_km;
```

## 🌍 Календарь сближений (`approaches/`)

Для каждого сближения страница дополнительно запрашивает данные об астероиде (диаметр, класс орбиты), чтобы показать больше информации.

## ⚠️ Сводка угроз (`threats/`)

### Что на ней есть?
- **Карточки статистики** — всего угроз, по уровням риска
- **Таблица угроз** — с Туринской и Палермской шкалами
- **Фильтр по риску** — все/высокий/средний/низкий

### Шкалы опасности

#### Туринская шкала (0-10)
| Уровень | Цвет | Что означает |
|---------|------|--------------|
| 0 | 🟢 Зелёный | Нет риска |
| 1-3 | 🟡 Жёлтый | Требует внимания |
| 4-7 | 🔴 Красный | Серьёзная угроза |
| 8-10 | 🟣 Бордовый | Неизбежное столкновение |

#### Палермская шкала
- **Отрицательная** (например, -2.5) — риск ниже фонового уровня
- **Положительная** — риск выше фонового

### Особенность
Если угроз нет вообще, страница показывает зелёный баннер: «✅ Угроз не обнаружено — это нормально!»

## 🛠️ Как добавить новую страницу?

### Инструкция для новичков

**1. Создайте папку**
```
frontend/my-new-page/
```

**2. Создайте 3 файла**
- `my-new-page.html` — разметка
- `my-new-page.css` — стили
- `my-new-page.js` — логика

**3. Подключите общие файлы**
В HTML добавьте в `<head>`:
```html
<link rel="stylesheet" href="../main.css">
<link rel="stylesheet" href="my-new-page.css">
```

В конце `<body>`:
```html
<script src="../api.js"></script>
<script src="../main.js"></script>
<script src="my-new-page.js"></script>
```

**4. Добавьте ссылку в меню**
Найдите в других HTML файлах блок `<nav>` и добавьте:
```html
<li><a href="my-new-page/my-new-page.html">Моя страница</a></li>
```

## 📝 Соглашения (как писать код)

### Имена функций
- `load...()` — загрузить данные (`loadAsteroids`)
- `render...()` — показать на экране (`renderTable`)
- `update...()` — обновить (`updateStats`)
- `filter...()` — отфильтровать (`filterByRisk`)
- `get...()` — получить/вспомогательная (`getRiskClass`)

### Имена переменных
- `all...` — все данные (`allAsteroids`)
- `filtered...` — после фильтрации (`filteredAsteroids`)
- `current...` — текущее значение (`currentSort`)

### Обработка ошибок
```javascript
try {
    // Пытаемся сделать что-то
    const data = await api.getAllAsteroids();
} catch (error) {
    // Если ошибка — показываем сообщение
    console.error('Ошибка:', error);
    showTableError('Не удалось загрузить данные.');

```

## ❓ Частые вопросы

### Почему не React/Vue?
Проект учебный. Чистый JavaScript проще для понимания новичками.

### Где хранятся данные?
Данные приходят с сервера (`http://localhost:8000`), который берёт их из базы данных PostgreSQL.

### Как обновить данные?
Данные обновляются автоматически при загрузке страницы. На бэкенде есть скрипт, который раз в 24 часа загружает свежие данные из NASA API.

### Почему таблица показывает только 15 астероидов?
На главной странице — для скорости загрузки. Полный каталог доступен на странице `/asteroids/`.
